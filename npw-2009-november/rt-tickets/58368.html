I ran into a related problem today, which may help narrow this down.
It appears that using *any* module with a nested namespace segfaults,
regardless of the contents of the module, but not if you omit the end-
of-statement semicolon. Very strange.

Ran lots of tests, and there's a GC component to the failure. Assume
that Foo and Foo::Bar are dummy modules, only containing the
string "1;" so they return true.

This works (as of r31495 on debian):
parrot perl6.pbc -e 'use Foo'

This works:
parrot perl6.pbc -e 'use Foo;'

This works:
parrot perl6.pbc -e 'use Foo::Bar'

This segfaults (trailing semicolon):
parrot perl6.pbc -e 'use Foo::Bar;'

This segfaults (with --gc-debug):
parrot perl6.pbc --gc-debug -e 'use Foo::Bar;'

This works (with -G):
parrot perl6.pbc -G -e 'use Foo::Bar;'

The backtrace is long and nasty, so I've attached as a separate file.
Hopefully others can reproduce and this helps someone with a better
understanding of the internals than I have.

-jeff
