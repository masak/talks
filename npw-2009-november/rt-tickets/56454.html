The source of the program (dumb fibonacci numbers) is as follows:

use v6;

sub fib( $n ){
if ( $n < 2 ) {
$n;
} else {
fib($n-1)+fib($n-2);
}
}

say fib(11);

it dies with segmentation fault if run without -G option:

./parrot languages/perl6/perl6.pbc fib
Segmentation fault

I have generated the corresponding PIR like this:
./parrot languages/perl6/perl6.pbc --target=pir --output=fib.pir fib

In order to run the PIR code I have added load_bytecode 'languages/perl6/perl6.pbc'
in the first sub and I have deleted the ':instanceof("Perl6Sub")' declaration for
sub "fib".

When I run it in parrot it works as expected. But If I change the requested
fibonacci number to 12 it starts to segfault again. If run with -G option it works.

Attanched is the PIR source
