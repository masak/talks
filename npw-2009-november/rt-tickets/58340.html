The attached fibonacci program seems to have some strange behavior in the
loop. There are two variables, $i, and $j, which contain the state, and $t
which saves $i temporarily.
In the first iteration of the loop, the output is
0 1 1

which is correct.

But the second iteration produces
2 2 2
instead of
1 1 2

And the rest are:
4 4 4
8 8 8
16 16 16
32 32 32
64 64 64
128 128 128
256 256 256
512 512 512
1024 1024 1024

which all show the same incorrect behavior as the second iteration. It is
almost as if the assignment statements, the second time through start
binding variables together.

Stephen Simmons

Here is a pair of related examples of variables being tied/bound together inappropriately.
First, swap.p6, which exhibits a similar problem.

sully:experiment stephensimmons$ perl6 swap.p6
ghi dhi lia qfw ast vzc ens lpf
ghi dhi
ghi dhi (ghi)
dhi dhi (dhi) # should be "dhi dhi (ghi)"
dhi dhi (dhi) # should be "dhi ghi (ghi)"
dhi dhi lia qfw ast vzc ens lpf

This has the advantage of not involving loops. However, the second program, swap_raw.p6
works correctly:

sully:experiment stephensimmons$ perl6 swap_raw.p6
1 2
1 2 (1)
2 2 (1)
2 1 (1)

Putting the working code back into a subroutine we get (swap_sub.p6):
sully:experiment stephensimmons$ perl6 swap_sub.p6
1 2
1 2
1 2 (1)
2 2 (1)
2 1 (1)
2 1

This is almost identical to the swap.p6 program, except for the type of the arguments being
passed in. Switching back to strings in swap_str.p6, we get:

sully:experiment stephensimmons$ perl6 swap_str.p6
dhi ghi
dhi ghi
dhi ghi (dhi)
ghi ghi (dhi)
ghi dhi (dhi)
ghi dhi

which again works correctly.

Not sure if this is the exact same problem as seen in the original fibonacci.p6, but it is at
least similar.

Stephen Simmons

As of r32441 the fibonacci.p6 program appears to be working properly:

$ cat fibonacci.p6
sub fibonacci($limit) {
my Int $i=0;
my Int $j=1;
say $i;
say $j;
while $j <= $limit {
my $t = $i;
$i = $j;
$j += $t;
say "$t $i $j";
}
}

fibonacci(1000);
